Note⠿
Issue #141 checkpoint:
Both max_tokens and max_completion_tokens survive decode + validation. The drop happens where non-OpenAI providers build their outbound payloads from req.MaxTokens only.
Provider-blind slot⠿

Normalize before provider resolution?

handler/chat.go:21–53 or gateway.go:422–486
  • At this point the request is just providers.Request
  • No final provider is known for fallback/load-balance/cost strategies
  • Can coalesce globally: if MaxTokens == nil, copy MaxCompletionTokens
  • Cannot honestly say "non-OpenAI only" here unless the rule is provider-blind

Bug site⠿

Non-OpenAI provider payload mapping

providers/*/*.go
  • Most providers build outbound structs from req.MaxTokens
  • Examples: Mistral, Anthropic, Bedrock, Gemini, Replicate
  • MaxCompletionTokens is already on the request but is not read
  • OpenAI non-streaming marshals the whole request; streaming uses SDK params
Design note⠿
Answer to "before provider resolution":
If the rule must be non-OpenAI only, it cannot be fully provider-blind. Put it after lookup / before Complete, in a provider wrapper, or inside each provider mapper. If global coalescing is acceptable, do it once before Route/RouteStream calls into provider selection.
